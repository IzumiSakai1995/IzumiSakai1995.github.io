函数式接口

如果一个接口只有一个抽象方法，那么该接口就是一个函数式接口

如果我们在某个接口上声明了@FunctionalInterface注解，那么编译器就回按照函数式接口的定义来要求该接口

如果一个接口声明了一个抽象方法，但是该抽象方法重写了java.lang.Obeject类的一个公有方法，对于函数式接口不影响。因为接口的任意一个实现都会有一个来自于 java.lang.Obeject或其他地方实现。

函数是接口的实例可以通过lambda表达式，方法引用，或者构造方法引用来创建。

optional类

描述

这是一个可以为Null的容器对象，如果值存在则isPresent()方法会返回true，调用get()方法会返回该对象

源码

基于值的类

  • 是final的和不可变的(可能包含指向可变对象的引用)
  • 有equals，hashcode,toString的实现，但基于实例本身的状态来实现，而不是根据其他对象或变量
  • 不会用实例之间的equality(==),hashcode,或者实例内在的锁
  • 判断equals仅根据equals()方法，而不是euqality(==)
  • 不包含可访问的构造方法，实例的创建通过工厂方法来创建的。工厂方法不保证返回实例的一致性

默认方法

Java 8 新增了接口的默认方法

简单说，默认方法就是接口可以有实现方法，而且不需要实现类去实现其方法

只需要在方法名前面加个default关键字即可实现默认方法。一方面保证老版本方法兼任，同时加入新的特性。

超越java8

科里化

科里化是一种将具备两个参数（比如x和y）的函数f转化为使用一个参数的函数g，并且这个函数的返回值也是一个函数，它会作为新函数的一个参数，后者的返回值和初始函数的返回值相同，即f(x,y)=(g(x))(y)。表示一种将一个带有n元组参数的函数转换成n个一元函数链的方法。

持久化数据结构

函数式方法不允许修改任何全局数据结构或者任何作为参数传入的参数。因为一旦对这些数据结构进行修改，两次相同的调用就很可能产生不同的结果，这违背了引用透明性的原则，我们也就无法将方法简单地看作由参数到结果的映射。
